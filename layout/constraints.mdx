
Constraints define **how elements behave when their container changes size**.

Instead of fixing an element to a specific position, constraints let you specify:

- Which edges an element is attached to
- Whether it resizes or stays fixed
- How it responds when the screen grows or shrinks

Constraints answer a simple question:

**What should happen to this element when the layout changes?**

### Why this matters

Constraints become important once layouts stop being static.

As soon as designs need to:

- Adapt to different screen sizes
- Handle dynamic content
- Be handed off for implementation

Without constraints, manual positioning starts to break down, resizing a frame can cause elements to drift or overlap, loosing alignment.
